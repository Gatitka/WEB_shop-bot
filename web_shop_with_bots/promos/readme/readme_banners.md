# Banner System — Technical README

## Назначение

Модуль Banner предназначен для управления рекламными баннерами
на сайте ресторана.

Поддерживает:

- карусель баннеров на главной странице;
- popup / modal окна;
- переходы к блюдам;
- переходы к категориям;
- внутренние promo-страницы;
- внешние ссылки;
- мультиязычные изображения;
- расписание показа;
- приоритеты отображения.

---

# Архитектура

## Основная модель

```python
Banner
```

Главная сущность системы баннеров.

---

# Где показываются баннеры

Баннеры используются:

- в мобильной карусели на главной странице;
- в desktop-карусели (если включено);
- как точки входа в promo-механики.

---

# Логика работы

Пользователь:

1. видит карточку баннера;
2. нажимает на баннер;
3. система выполняет действие в зависимости от `action_type`.

---

# Action Types

## DISH

```python
action_type = 'dish'
```

Открывает карточку блюда.

Используется поле:

```python
dish
```

---

## CATEGORY

```python
action_type = 'category'
```

Открывает категорию меню.

Используется поле:

```python
category
```

---

## PROMO_PAGE

```python
action_type = 'promo_page'
```

Открывает внутреннюю promo-страницу сайта.

Используется поле:

```python
promo_slug
```

Пример:

```text
birthday
```

→

```text
/promo/birthday/
```

---

## EXTERNAL

```python
action_type = 'external'
```

Переход на внешний URL.

Используется поле:

```python
external_url
```

---

## MODAL_SVG

```python
action_type = 'modal_svg'
```

Открывает popup / modal окно поверх страницы.

Поддерживаются:

- SVG
- WEBP
- PNG
- JPG

Используется для:

- условий акций;
- подарков;
- информации о доставке;
- birthday promo;
- рекламных постеров.

---

## NONE

```python
action_type = 'none'
```

Некликабельный баннер.

Используется как декоративный или информационный элемент.

# Изображения баннеров

## Основное изображение

```python
image
```

Используется как дефолтное изображение баннера.

---

## Языковые версии

```python
image_ru
image_en
```

Если отсутствуют —
используется дефолтный `image`.

---

# Требования к изображениям

## Форматы

Разрешены:

- jpg
- jpeg
- png
- webp

---

## Автоконвертация

JPG/PNG автоматически конвертируются в WEBP
при сохранении модели.

---

## Размеры

Рекомендуется:

```text
600×600 px
```

Минимум:

```text
400×400 px
```

---

## Пропорции

Квадрат:

```text
1:1
```

Допуск:

```text
±15%
```

---

## Вес

Максимум:

```text
500 KB
```

# Popup / Modal контент

## Поля

```python
modal_svg
modal_svg_ru
modal_svg_en
```

Несмотря на название, поддерживают:

- SVG
- WEBP
- PNG
- JPG

---

# Когда использовать SVG

SVG рекомендуется использовать для:

- текстовых promo;
- условий акции;
- схем;
- простого векторного дизайна;
- минималистичных popup.

Преимущества:

- маленький размер;
- идеальная резкость;
- масштабируемость.

---

# Когда использовать WEBP

WEBP рекомендуется использовать для:

- food posters;
- фотографий еды;
- сложных фонов;
- рекламных афиш;
- коллажей.

Преимущества:

- меньше вес;
- лучше для фотографий;
- стабильнее рендерится на mobile.

---

# Приоритеты

Поле:

```python
priority
```

Чем меньше число —
тем раньше баннер показывается в карусели.

Пример:

```text
1 → показывается раньше 2
```

---

# Active Period

## Поля

```python
active_from
active_until
```

Используются для автоматического ограничения периода показа баннера.

---

# Мультиязычность

Поддерживаются:

- sr-latn
- ru
- en

Логика fallback:

```text
ru/en отсутствует
↓
используется sr-latn
```

---

# Копирование баннеров

В админке доступна функция:

```text
Создать копию
```

Что делает:

- создаёт новый Banner;
- копирует изображения;
- копирует popup файлы;
- делает баннер неактивным;
- автоматически назначает новый priority.

# Рекомендации дизайнеру

## Баннер-карточка

### Safe zone

Важный контент должен находиться
в центральной области ~70%.

---

## Popup

Для popup рекомендуется:

- вертикальный формат;
- крупный текст;
- минимум мелких деталей;
- высокая контрастность.

---

# Recommended Popup Sizes

## WEBP

```text
900×1200 px
```

или

```text
1080×1920 px
```

---

## SVG

Любой размер,
но рекомендуется artboard:

```text
1080×1920
```

---

# Performance

## Banner images

Автоматически конвертируются в WEBP.

---

## Popup SVG

Не рекомендуется использовать:

- тяжелые blur;
- filters;
- сложные masks;
- огромные embedded images.

---

# Frontend Notes

Frontend должен:

- lazy-load popup content;
- preload first banners;
- блокировать body-scroll при modal;
- поддерживать swipe карусели;
- закрывать popup по backdrop/cross.
